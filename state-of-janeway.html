Release

v1.8.0 — Paris.

The biggest release in years. Three platform-level shifts plus a long tail of polish, accessibility, and bug fixes.

  • ROR integration — affiliations are now first-class, registered, exportable data.
  • CRediT Taxonomy — structured contributor roles across submission, display, and metadata export.
  • Typesetting in core — the standalone plugin is folded into the platform and deprecated.
v1.8.0-RC-11 · "Paris"09

v1.8 · Highlight

ROR — affiliations, finally first-class.

  • New Organization model with a RORStatus lifecycle, plus OrganizationName, RORImport and RORImportError for audit trails.
  • ROR data imported on install and refreshed monthly; atomic, chunked bulk imports; MySQL-safe; batch size is a configurable setting.
  • Authors can pull ROR-controlled affiliations directly from ORCiD during submission.
  • ROR identifiers flow into JATS and Crossref 5.4.0 exports.
  • Readers see ROR affiliations on the article and public profile pages.
  • Author / librarian RST documentation shipped.

v1.8 · Hidden gems

The bits that don't make the headline.

  • Triple-anonymous review. Section editors can no longer see author PII until review is complete — new security decorator, template tags, and PII-stripped email templates. Triple-anonymous is now a built-in option.
  • Repository submission review page. Authors get a final summary of metadata, authors and files (with back-links to fix mistakes) before their preprint goes in.
  • Editors control prepub notifications. Different recipients per form, full To / CC / BCC visibility, logged in the notification history.
  • Italics in titles and content fields. Proper italicisation in journal titles, article titles and other content — safe HTML storage in the database, not bleach-stripped.
  • i18n investment. Multilingual news items, translatable news display fields, and full Spanish language support.
  • Conditional accept / desk-rejection emails. Templates fire per decision type automatically.
  • Human-readable dates front-of-house. "3 weeks ago" instead of bare timestamps.

v1.9 · In flight

Accessibility, made measurable.

The audit, in code

  • Every WCAG 2.2 AA Success Criterion has been audited — ~50 audit commits, one per SC, against all three core themes.
  • Each SC has a documented level per theme: supports, partially supports, or not applicable.
  • A new update_a11y_conformance management command builds the conformance dataset from source — plugins included.
  • Every theme (and journal) now publishes its own VPAT, with audit links pointing to the PR that fixed each criterion.
  • Press-level a11y page rolls up VPATs for every journal underneath it.

Concrete fixes

  • Material keyboard navigation — full keyboard reachability across the Material theme.
  • Lightbox focus returns to its triggering element on close.
  • Page language follows translation — <html lang> updates as the UI language changes, including in print templates.
  • Full-text-search pages, button semantics, pagination arrows, endnote backlinks, nav landmarks, list markup, reader page structure — all audited and fixed.
  • Per-theme dyslexia / text-size controls landed in Clean and Material.

Accessibility is no longer a one-off audit. It's a per-theme, version-stamped artefact — published on every site.

New theme

Clarity.

A more modern theme for Janeway, with a switchable colour-palette system and a long list of accessibility improvements over the Clean theme.

  • Four built-in palettes — evergreen (default), ocean, cardinal, and paper (white header for logo-led journals). Switchable per journal from Manager › All Settings.
  • Built on CSS custom properties, so the custom styling plugin can override individual variables without forking the theme.
  • Includes a full repository (preprint) front-end, ready to enable in REPOSITORY_THEMES.
  • Accessibility improvements over Clean.
  • Dark mode coming soon — a full WCAG-compliant dark palette is in development.
  • Merging into core — Clarity will land in Janeway core and become the default theme for new journals.

Plugin spotlight · funded by UCL Press

Isolinear — preprints & articles, wired together.

  • Publish a preprint from the journal workflow. At Editor Assignment, an editor can approve an article for preprint publication — Isolinear creates a Preprint in a headless Janeway repository, inheriting title, abstract, authors, keywords and licence from the article.
  • Versions, end to end. During Review, editors can push a revised manuscript as a new PreprintVersion — same record, traceable history, same DOI lineage.
  • PDF with a cover sheet. DOCX / RTF / ODT manuscripts are converted via Pandoc + XeTeX, then spliced together with a templated cover sheet on every version.
  • Published peer reviews ride along with the preprint — readers get the article, its versions, and the reviewer reports in one place.
  • Every preprint version gets its own Crossref DOI — distinct from the journal article, deposited automatically.
  • Coming soon: an editorial history view — the full submission, review and publication timeline surfaced on the preprint record.

Core feature · funded by Iowa State University Digital Press

Linking books with preprints — OERs, first-class.

  • Open Educational Resources — workbooks, exercises, datasets, supplementary materials — used to live inside book chapters as nameless files. They now get their own preprint record.
  • Each OER has its own DOI, abstract, authors, licence, downloads, and metrics — independently citable, independently versioned.
  • Linked Items. A new mechanism wires the OER back to its parent book or chapter — discoverable from both ends, citable in either direction.
  • Built into core Janeway.

Governance

The Janeway Publishing Technologies Board.

The JPTB is a strategic advisory body of the Open Library of Humanities. It provides long-term vision and expert guidance for the development of Janeway, and acts as a forum for discussion that helps keep the platform responsive to its global community of users and contributors.

  • Strategic advice — on direction, technical priorities, and platform sustainability.
  • Meetings — twice annually, to review progress, give strategic input, and make advisory recommendations.
  • Reporting — the Director of Publishing Technology updates the Board regularly; the Board may produce recommendations or strategy notes to inform platform planning. The governance structure is reviewed annually.
  • Membership — publishing professionals, librarians, and open-source advocates serving staggered, renewable three-year terms. A Chair is elected as needed to facilitate meetings and build consensus.
  • Sits alongside the OLH Academic Advisory Board and Library Board within the wider OLH governance.
